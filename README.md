# Hotelium PMS User Manual

This site is the **Hotelium PMS User Manual**, built with [Docusaurus](https://docusaurus.io/). The built site is published at **https://docs.hotelium.com.mm** (see `docusaurus.config.js`).

## Requirements

- **Node.js** ≥ 20 (see `package.json` → `engines.node`)

## Quick start

```bash
npm install
npm run start
```

Open the browser; most changes reload automatically.

## Scripts

| Command | Purpose |
|---------|---------|
| `npm run start` | Local dev server |
| `npm run build` | Build static site (fails on broken links) |
| `npm run serve` | Serve the `build` folder locally (test production build) |
| `npm run validate` | Same as `npm run build` — use in CI or before pushing |
| `npm run deploy` | Build and deploy (e.g. GitHub Pages) |

## Project structure (pmsusermanual)

| Path | Purpose |
|------|---------|
| `docs/` | Manual content (Markdown). Quick start, intro, operations, configuration, general, technical. |
| `src/` | React components and styles (homepage, custom CSS). |
| `static/` | Images, favicon, logo, `.nojekyll`, custom JS (e.g. back-to-top). |
| `docusaurus.config.js` | Site title, URL, theme, navbar, footer, docs options. |
| `sidebars.js` | Sidebar is autogenerated from `docs/`; use `sidebar_position` in frontmatter. |

## Git and .gitignore

The repo uses a **`.gitignore`** so these are not committed:

- **`/node_modules`** — Dependencies (reinstall with `npm install`)
- **`/build`** — Production build output
- **`.docusaurus`**, **`.cache-loader`** — Docusaurus cache
- **`.env*`** — Environment and secrets
- **Logs** — `npm-debug.log*`, `yarn-error.log*`, etc.
- **OS/IDE** — `.DS_Store`, `Thumbs.db`, `.idea`, `.vscode` (optional)

After cloning, run `npm install` once. To build with “Last updated” dates from git, ensure the project is in a git repo and `showLastUpdateTime: true` in `docusaurus.config.js`.

## Pushing to a remote

1. Create a new empty repo on GitHub (or GitLab, etc.).
2. Add the remote and push:

   ```bash
   git remote add origin https://github.com/YOUR_USERNAME/REPO_NAME.git
   git branch -M main
   git push -u origin main
   ```

3. For GitHub Pages deploy: `USE_SSH=true npm run deploy` or `GIT_USER=<username> npm run deploy`.

## CI and links

- **Build = link check:** `npm run build` fails if any doc link is broken.
- **GitHub Actions:** A **Docs** workflow runs on push/PR to `main` (or `master`), runs `npm ci` and `npm run build`. See [.github/workflows/docs.yml](.github/workflows/docs.yml).

## Maintenance

- **Technical and maintenance** — In the built site, see **Docs → Technical & Maintenance → Maintenance** (link style, CI, must-have screenshots, versioning).

## Deployment

**GitHub Pages (SSH):**
```bash
USE_SSH=true npm run deploy
```

**GitHub Pages (HTTPS):**
```bash
GIT_USER=<Your GitHub username> npm run deploy
```

This builds the site and pushes the `build` output to the `gh-pages` branch (or your configured branch).
