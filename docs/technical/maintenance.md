# Technical and Maintenance

Guidelines for keeping the Hotelium PMS manual buildable, link-valid, and consistent.

## Link Style

Use **absolute paths** for doc-to-doc links so they work from any page and from the built site:

- **Do:** `[Introduction](/docs/intro)`, `[Reservations](/docs/operations/reservations/overview)`
- **Avoid:** `[Introduction](../intro)`, `[Reservations](./reservations)` (relative paths)

Internal anchors are fine: `[Section](#section-name)`.

## Broken Links and CI

- **Build = link check:** `npm run build` fails if any doc link is broken (`onBrokenLinks: 'throw'` in `docusaurus.config.js`).
- **CI:** If GitHub Actions is used, the **Docs** workflow runs on push/PR to `main` (or `master`) and runs `npm ci` and `npm run build`. Fix broken links before merging.
- **Local:** Run `npm run build` (or `npm run validate`) before pushing to catch broken links early.

## Versioning (Future)

If you need to ship multiple manual versions (e.g. v1.0, v1.1), use [Docusaurus versioning](https://docusaurus.io/docs/versioning):

- Add a version, e.g. `docusaurus docs:version 1.0`.
- Docs are versioned under `/docs/1.0/...`; current is `/docs/...`.
- Document your release process (when to cut a version, how to update the dropdown).

## Scripts Reference

| Command | Purpose |
|---------|---------|
| `npm ci` | Clean install (use in CI). |
| `npm run build` | Build site; fails on broken links. |
| `npm run validate` | Same as `npm run build` (alias for CI or pre-commit). |
| `npm run start` | Local dev server. |

## File Layout

- **docs/** — Markdown source; sidebar is autogenerated from here. Use `sidebar_position` in frontmatter and `_category_.json` for order.
- **static/** — Served at `/`. Put screenshots in `static/img/docs/`, scripts in `static/js/`.
- **src/** — React components, CSS, theme overrides.
- **docusaurus.config.js** — Title, theme, navbar, footer, plugins (search, etc.).
- **sidebars.js** — Sidebar config (currently autogenerated from `docs/`).
