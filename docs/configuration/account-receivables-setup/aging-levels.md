# Aging Levels

**Aging Levels** define the time buckets used in AR aging reports to categorise outstanding balances by how long they have been overdue.

## Access

**Configuration → Account Receivables → Aging Level**

## Fields

| Field | Description |
|-------|-------------|
| Name | Label for the aging bucket (e.g. Current, 1–30 Days, 31–60 Days, 61–90 Days, Over 90 Days). |
| Days From | Start of the aging period in days. |
| Days To | End of the aging period in days. |
| Active | Whether this level is used in reports. |

## Usage

Aging levels appear in AR aging reports to show how long invoices or balances have been outstanding. Define at least one level for current (0 days) and additional levels for overdue periods.
